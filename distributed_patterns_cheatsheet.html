架构
📖 Saga 长事务 🔴
Saga
 订外卖：下单→付款→商家接单→配送。商家不接单则退款取消。 将跨服务事务拆分为多个本地事务，失败时按相反顺序执行补偿。 电商订单旅游预订金融转账长流程
架构
📜 事件溯源 🔴
Event Sourcing
 银行账本：不记余额，只记每笔存取，余额是算出来的。 只存储状态变化的事件序列，当前状态通过重放事件推导。 审计追踪状态重建时间旅行
架构
⚡ CQRS 读写分离 🔴
Command Query Separation
 餐厅：厨师做菜（写），服务员上菜（读），分工明确。 将读和写操作分离到不同模型，各自独立优化。 高并发读写复杂查询报表系统
架构
🚪 API 网关 🔴
API Gateway
 酒店前台：客人来了先找前台，前台再分配房间、处理登记。 为微服务集群提供统一入口，负责路由、认证、聚合。 微服务入口统一认证流量聚合
架构
🚌 事件总线
Event Bus
 公司公告栏：有人发布消息，所有人都能看见，不用一个个通知。 通过事件驱动实现服务间通信，发布者发布，订阅者异步消费。 服务间通信状态变更松耦合
架构
🗄️ 服务独享数据库
Database per Service
 每家公司有自己的账本，互不干涉。查账只能找对方公司。 每个微服务拥有独立数据库，不与其他服务共享数据存储。 微服务拆分数据自治技术异构
架构
🔪 数据分片
Sharding
 图书馆按字母分书：A-F放一楼，G-L放二楼，不用在一层楼找所有书。 将大数据集按规则（哈希/范围）拆分到多个节点。 大表拆分水平扩展分布式数据库
架构
🥪 分层缓存
Cache Hierarchy
 办公室：桌上→抽屉→文件柜，常用文件放最近处。 缓存分为多个层级（L1/L2/L3），热点数据放在最近层。 RedisCDN热点数据性能加速
代码
🔌 熔断器 🔴
Circuit Breaker
 电路保险丝：电流过载时自动断开，防止烧毁电器。 当服务调用失败率达到阈值时，自动断开连接，防止故障扩散。 微服务调用第三方API容错
代码
🚦 限流 🔴
Rate Limiting
 高速收费站：每秒只放行固定数量车辆，防止拥堵。 控制请求速率或并发数，防止系统过载。 API网关接口防刷抢购
代码
🔄 重试 🔴
Retry
 打电话对方占线，过一会儿再打，不是永远等下去。 调用失败时按策略重试，避免偶发故障影响整体。 网络请求消息发送自动恢复
代码
⏱️ 超时 🔴
Timeout
 微波炉定时：设10分钟，到时间就自动停止，不会无限加热。 为每个远程调用设置最大等待时间，超时后立即返回。 RPC调用HTTP请求防止阻塞
代码
🛳️ 舱壁隔离 🔴
Bulkhead
 轮船防水舱：一个舱进水，其他舱不受影响，船不会沉。 将资源划分为独立池，局部故障不会影响整体系统。 线程池隔离服务分组故障隔离
代码
🔽 降级
Degradation
 医院急诊：病人太多时，优先救治危重病人，轻症暂缓。 系统过载时关闭非核心功能，保证核心业务可用。 大促活动服务过载有损服务
代码
♻️ 幂等 🔴
Idempotency
 电梯按钮：按一次和按十次效果一样，电梯只来一趟。 重复执行同一操作不会产生副作用，保证重试安全。 支付回调接口重试防止重复
代码
🔐 分布式锁
Distributed Lock
 公共厕所：有人进去就锁门，外面的人必须等待。 在分布式环境下实现互斥访问，防止多个进程同时操作共享资源。 库存扣减订单防重互斥访问
运维
🔍 服务发现 🔴
Service Discovery
 医院导诊台：新医生来了登记，病人来了问导诊台找哪个医生。 服务启动时注册自己，客户端通过服务名动态发现可用实例。 微服务注册动态扩缩容K8s
运维
⚖️ 负载均衡 🔴
Load Balancing
 超市收银台：多个柜台分流顾客，避免某个柜台排太长队。 将请求均匀分配到多个服务实例，避免单点过载。 网关分发服务集群高可用
运维
💓 健康探针 🔴
Health Probe
 体检中心：定期给人做检查，有问题早发现早治疗。 定期检测服务健康状态，不健康实例从流量中剔除。 K8s存活检查自动重启流量摘除
运维
🔁 调和回路
Reconciliation Loop
 扫地机器人：设定目标（全扫一遍），不断调整路径直到完成。 持续比较实际状态与期望状态，自动执行操作使两者对齐。 K8s控制器配置修复自动对齐
运维
🐦 金丝雀发布
Canary Release
 煤矿工人带金丝雀下矿：先放小鸟探路，安全了人再下去。 新版本先发布给少量用户，验证无问题后再逐步扩大流量。 版本发布A/B测试风险控制
运维
📋 GitOps
GitOps
 装修图纸：设计师画好图纸，工人按图施工，装修完跟图纸一致。 以 Git 为唯一真相来源，配置变更自动同步到运行时。 K8s部署基础设施管理配置即代码
运维
🔬 分布式追踪 🔴
Distributed Tracing
 快递跟踪：从发货到签收，每个中转站都扫码记录，全程可追踪。 为请求分配唯一 Trace ID，串联全链路调用，定位延迟和故障。 性能分析故障定位链路追踪
运维
📝 结构化日志
Structured Logging
 表格登记：不是手写笔记，而是统一表格，方便分类查找。 使用统一键值对格式（JSON）记录日志，便于机器解析和分析。 日志聚合ELK/Loki可检索
运维
📜 声明式配置
Declarative Config
 点外卖：你说"我要一份宫保鸡丁"，不用管厨房怎么炒。 描述系统的期望状态，由系统自动执行操作达成目标。 K8s YAMLTerraform自动化部署
架构
🏍️ Sidecar / 服务网格 🔴
Sidecar / Service Mesh
 酒店房间旁边的管家房：主房住客人，管家房住服务员，互不影响。 将日志、监控、安全等非业务功能从主服务抽离，放到独立 Sidecar 代理中统一管理。 IstioLinkerdEnvoy服务网格
